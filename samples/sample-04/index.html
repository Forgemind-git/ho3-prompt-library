Ticket Triage & Priority

EXAMPLE
C — Context
I'm a support team lead. We receive a mix of tickets — bugs, billing issues, feature requests, and simple how-to questions. Agents spend too long deciding priority instead of resolving tickets.
R — Role
Act as an experienced support operations manager who has built triage systems for high-volume SaaS helpdesks.
I — Instruction
Read this support ticket and return a triage decision: category, priority level, and the first action the agent should take.
S — Spec
Return three fields only: Category (Bug / Billing / Feature Request / How-To / Other), Priority (P1/P2/P3), First action (one sentence max 20 words). No prose.
P — Performance
A great triage takes under 5 seconds to read and leaves zero ambiguity about what the agent does next.
