# Prompt 06 — SEO Meta Description

**Library:** Content Marketer Prompt Library  
**Role:** Content Marketing Manager

---

## Purpose

Write SEO-optimised meta descriptions and title tags that stay within Google's character limits, include the target keyword naturally, and are written to earn a click — not just describe the page.

---

## CRISP Breakdown

| Element | Content |
|---------|---------|
| **Context** | Page: [URL or page description]. Target keyword: [primary keyword]. Search intent: [what the searcher wants — informational / commercial / navigational]. Current meta (if any): [paste current meta to improve]. |
| **Role** | You are an SEO specialist who writes meta descriptions that treat the search result page as a conversion moment — you're competing against 9 other results and every word counts. |
| **Instruction** | Write 3 variations each of: title tag and meta description. Each variation must include the target keyword, stay within character limits, and use a different click-trigger (curiosity, specificity, urgency, benefit). |
| **Spec** | Title tag: 50–60 characters max. Meta description: 150–160 characters max. Keyword in first 30 characters of title tag. 3 variations of each. Character count shown next to each. |
| **Performance** | All 6 pieces stay within character limits (checked). Keyword is in all 6 pieces. At least one variation for each uses a different click motivator. A/B testable. |

---

## Full Prompt

```
Context: I'm writing SEO meta tags for a page at [URL or description of page content]. 
- Target keyword: [primary keyword]
- Search intent: [informational / commercial / navigational / transactional]
- Page content summary: [2–3 sentences describing what the page covers]
- Brand name to include: [company name — include in title tag or not?]
- Current meta description (if updating): [paste current version or "none"]
- Top competitor title/meta example (optional): [paste a competitor's to differentiate from]

Role: Act as an SEO specialist who treats the search result page as a conversion moment. Every character in a title tag and meta description is competing against 9 other results — you write to earn the click, not just describe the page.

Instruction: Write 3 variations of each:
1. Title Tag — 3 variations (50–60 characters each)
   - Variation A: Leads with the keyword, benefits-focused
   - Variation B: Question or "how to" format
   - Variation C: Specificity/number format
2. Meta Description — 3 variations (150–160 characters each)
   - Variation A: Benefit-led
   - Variation B: Problem/solution format
   - Variation C: Urgency or specificity

Spec:
- Title tag: 50–60 characters max (show character count in brackets)
- Meta description: 150–160 characters max (show character count in brackets)
- Keyword must appear in all 6 pieces
- Each variation uses a different click motivator (not the same opening word)
- Show character count next to each piece

Performance bar: All pieces are within character limits. Keyword is in all 6. Each pair of title + meta variation works together. A/B testable — variations are meaningfully different, not just word-swap.
```

---

## Before Example (Weak Prompt)

**Prompt used:**
> Write a meta description for our customer support software page.

**Output produced:**
> "We offer customer support software for businesses of all sizes. Our platform helps you manage tickets and improve customer satisfaction. Learn more today!"

**Why it's weak:** No keyword, 147 characters but no intent match, "businesses of all sizes" is the broadest possible audience claim, "learn more today" is the most generic CTA possible. Nobody clicks this over the 9 other results.

---

## After Example (With CRISP)

**Placeholders filled:**
- Page: Helply's main product page (helply.io/product)
- Keyword: "customer support software for small teams"
- Intent: Commercial (evaluating tools to buy)
- Page covers: features for centralised support, ticket management, AI automation, reporting

**Output produced:**

> **TITLE TAGS**
>
> Variation A (Benefit-focused):  
> `Customer Support Software for Small Teams | Helply` [49 chars]
>
> Variation B (Question format):  
> `Managing Support Without a Big Team? Try Helply` [47 chars]
>
> Variation C (Specificity):  
> `Handle 500+ Tickets/Week With a Team of 3 | Helply` [50 chars]
>
> ---
>
> **META DESCRIPTIONS**
>
> Variation A (Benefit-led):  
> `Helply gives small support teams the tools to handle high ticket volume without adding headcount. Centralised inbox, AI triage, live reporting.` [142 chars]
>
> Variation B (Problem/Solution):  
> `Support chaos eating your team alive? Helply centralises every ticket in one place — so a 3-person team can handle what used to need 8. Free trial.` [148 chars]
>
> Variation C (Specificity):  
> `Teams using Helply cut ticket volume by 40% in 60 days. Customer support software built for small teams who need to punch above their weight.` [141 chars]
>
> ---
>
> **Recommended Pairings for A/B Testing**
>
> - Test 1: Title A + Meta C (credibility pairing — name + stat)
> - Test 2: Title C + Meta B (specificity + problem/solution)
> - Test 3: Title B + Meta A (question hook + benefit payoff)
