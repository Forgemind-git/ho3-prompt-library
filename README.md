# HO3 — Prompt Library

> Hands-on portfolio project · **Week 2** · **Peer-reviewed** · module M6. Part of the **ForgeMind AI — AI Productivity Essentials** course.

## Goal

**Done when:** A documented, reusable prompt library (8–12 prompts)

## What to ship

A README index + the prompt files organised by use-case + a before/after example for each.

## Pick a problem statement

Choose **one** of these real use-cases — or bring your own (get it approved first):

1. You are an SDR rewriting outreach from scratch every day and the quality swings wildly. Build a documented pack of 8 to 12 CRISP prompts for the sales motion (cold email, follow-up, objection handling, meeting recap) for a named product and buyer. Success: each prompt has a before/after example proving the new version writes a better message.

2. As a solo content marketer you reinvent the wheel for every post. Build an organised pack of 8 to 12 CRISP prompts covering blog outlines, repurposing one piece to LinkedIn and X, SEO meta, and a newsletter, written for your niche. Success: a README index plus a before/after example per prompt showing the upgrade in output.

3. You are a recruiter and your job ads and screening notes are inconsistent. Build a documented pack of 8 to 12 CRISP prompts for hiring (JD writing, a resume-screening rubric, interview questions, candidate feedback) for one real role. Success: each prompt ships with a before/after example that proves the structured version is sharper.

4. Your support team answers the same issues in wildly different tones. Build an organised pack of 8 to 12 CRISP prompts for support (reply drafts by issue type, tone adjustment, escalation summaries) grounded in your product. Success: a README plus a before/after example per prompt showing cleaner, on-brand replies.

5. As a new people-manager you stare at a blank page before every 1:1 and review. Build a documented pack of 8 to 12 CRISP prompts for managing (1:1 prep, performance-review drafts, status updates, meeting agendas). Success: each prompt has a before/after example proving the structured version saves you time and reads better.
